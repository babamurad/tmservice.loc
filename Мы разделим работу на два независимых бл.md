## Блок 1: Backend API (Laravel 13+)

Здесь мы создаем базу данных и REST API. Никаких Blade-шаблонов, только JSON-ответы.

### Этап 1. База данных и модели (миграции)

- [ ] Шаг 1.1: Модель User
  - Поля: `id`, `phone` (string, unique), `password`, `role` (enum: `admin`, `master`, `client`), `remember_token`, `timestamps`.
  - Для первого администратора в сидерах можно использовать `babamurad2010@yandex.ru` как контактный email для восстановления, если позже добавится поле `email`. Для входа оставляем только телефон.

- [ ] Шаг 1.2: Модель City
  - Поля: `id`, `name_ru`, `name_tm`, `is_active` (boolean).
  - Сразу напиши Seeder и заполни городами: Туркменабад, Ашхабад, Мары, Дашогуз, Балканабад.

- [ ] Шаг 1.3: Модель Category
  - Поля: `id`, `name_ru`, `name_tm`, `icon_url`, `is_active`.
  - Напиши Seeder: Сантехник, Электрик, Ремонт авто, Сборка мебели.

- [ ] Шаг 1.4: Модель MasterProfile
  - Связь: `belongsTo(User)`.
  - Поля: `id`, `user_id` (foreign key), `city_id` (foreign key), `category_id` (foreign key), `bio` (text, nullable), `is_free` (boolean, default: `true`), `qr_code_path` (string, nullable).

- [ ] Шаг 1.5: Модель PortfolioImage
  - Связь: `belongsTo(MasterProfile)`.
  - Поля: `id`, `master_profile_id` (foreign key), `image_path` (string).

### Этап 2. Авторизация (Sanctum)

- [ ] Шаг 2.1: Установить и настроить Laravel Sanctum.
- [ ] Шаг 2.2: Создать `AuthController@register`
  - Валидация: `phone` (required, unique), `password`, `role`.
  - Логика: создает пользователя. Если роль `master`, сразу создает пустую запись в `master_profiles`. Возвращает `token` и данные пользователя.
- [ ] Шаг 2.3: Создать `AuthController@login`
  - Логика: проверяет `phone` и `password`. Возвращает `token`.
- [ ] Шаг 2.4: Создать `AuthController@logout`
  - Логика: удаляет текущий токен пользователя (требует middleware `auth:sanctum`).

### Этап 3. API-эндпоинты (контроллеры)

- [ ] Шаг 3.1: `DirectoryController` (публичные маршруты)
  - `GET /api/cities` — отдает все активные города.
  - `GET /api/categories` — отдает все активные категории.
  - Совет: реализуй кэширование (Redis/File) для этих запросов, чтобы реализовать offline-first подход в мобильном приложении и не дергать базу при каждом открытии приложения.

- [ ] Шаг 3.2: `MasterController` (публичные маршруты)
  - `GET /api/masters` — принимает параметры `?city_id=1&category_id=2`. Возвращает список профилей мастеров с пагинацией `paginate(15)`, сортируя сначала тех, у кого `is_free = true`.
  - `GET /api/masters/{id}` — возвращает детальную информацию о мастере, включая массив `portfolio_images` и телефон `user->phone`.

- [ ] Шаг 3.3: `ProfileController` (защищено `auth:sanctum`, только для мастеров)
  - `POST /api/profile/update` — обновляет `bio`, `city_id`, `category_id`.
  - `POST /api/profile/status` — переключает тумблер `is_free` (`true/false`).
  - `POST /api/profile/portfolio` — загружает картинку, сжимает ее через Intervention Image, сохраняет путь в `portfolio_images`.

### Этап 4. Генерация QR-кодов

- [ ] Шаг 4.1: Установить пакет `simplesoftwareio/laravel-qrcode`.
- [ ] Шаг 4.2: В `ProfileController@generateQr` написать метод, который генерирует картинку формата `.png`.
  - Содержимое QR-кода — ссылка вида `appscheme://master/{id}` (deep link для мобильного приложения).
  - Картинка сохраняется в `storage/app/public/qr`, а путь записывается в `master_profiles.qr_code_path`.

## Блок 2: Мобильное приложение (React Native / Expo)

Используем Expo, чтобы не возиться с нативными сборками на старте.

### Этап 5. Инициализация и навигация

- [ ] Шаг 5.1: Создать проект: `npx create-expo-app ServiceApp`.
- [ ] Шаг 5.2: Установить зависимости навигации: `@react-navigation/native`, `@react-navigation/bottom-tabs`, `@react-navigation/native-stack`.
- [ ] Шаг 5.3: Настроить роутинг
  - `Stack Navigation` для `LoginScreen` и `RegisterScreen`.
  - `Bottom Tabs` для авторизованной зоны: `CatalogTab`, `SearchTab`, `ProfileTab`.
- [ ] Шаг 5.4: Установить хранилище токенов `expo-secure-store`. Написать хук `useAuth`, который при старте приложения проверяет наличие токена и направляет либо на логин, либо на главную.

### Этап 6. Экраны клиента (поиск мастеров)

- [ ] Шаг 6.1: `CatalogScreen`
  - При монтировании делает `GET`-запрос к `/api/categories` и `/api/cities`.
  - Выводит выпадающий список городов сверху и сетку категорий в центре.

- [ ] Шаг 6.2: `MastersListScreen`
  - Открывается при клике на категорию.
  - Делает `GET`-запрос к `/api/masters?category_id=X&city_id=Y`.
  - Выводит карточки мастеров (имя, зеленая/красная точка статуса).

- [ ] Шаг 6.3: `MasterDetailScreen`
  - Показывает биографию и галерею работ.
  - Кнопка «Позвонить». Реализация: `Linking.openURL('tel:+993XXXXXXXX')` — вызывает системную звонилку.

### Этап 7. Экраны мастера (управление визиткой)

- [ ] Шаг 7.1: `MyProfileScreen`
  - Запрос к своему профилю.
  - Вывод тумблера «Свободен / Занят». При клике отправляется `POST`-запрос на `/api/profile/status`.
  - Для мгновенного обновления статуса у всех клиентов позже идеально впишется Laravel Reverb через WebSockets, но пока делаем обычный REST.

- [ ] Шаг 7.2: Загрузка портфолио
  - Установить `expo-image-picker`.
  - Сделать кнопку загрузки фото из галереи телефона.
  - При выборе фото отправляется `Multipart-form POST`-запрос на `/api/profile/portfolio`.

- [ ] Шаг 7.3: Экран `MyQRCode`
  - Делает запрос на backend, получает URL сгенерированной картинки QR-кода и выводит ее в компоненте `<Image>` на весь экран с максимальной яркостью, чтобы клиенту было удобно сканировать.
